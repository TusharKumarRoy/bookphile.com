MVC: 
    User visits URL → Route → Controller → Model (data) → View (HTML) → User sees page
    
    Component	 What it does	    File location
    
    Route	    Maps URL to code	>routes\web.php
    Controller	Handles logic	    >app\Http\Controllers
    Model	    Talks to database	>app\Models
    View	    HTML template	    >resources\views


5. creating a simple route: 

   <?php
    Route::get('/hello', function () {
    return 'Hello World!';                  // writing 'text to write' shows the text
});                                         // writing view('view name'); like view('hello'); look for >resources\views\hello.blade.php   opens the view or html page

passing data from routes to views .

    Route::get('/hello', function () {
    $sitename = 'bookphile.com';

updating in the view to access the variables:

creating a basic controller : look in the routes\web.php , we want routes to be clean as possible , so the functions in the routes are replaced with controller functions 
                            controller makes it easy to handle how information is passed in the routes

command : php artisan make:controller PageController    <- controller_name

>app\Http\Controllers\PageController.php

-- add this function 
    public function hello()
    {
        $sitename = 'bookphile.com';
        $tagline = 'A book lover\'s paradise';  
        return view('hello', compact('sitename', 'tagline'));
    }

-- modify >routes\web.php
   Route::get('/hello', [App\Http\Controllers\PageController::class, 'hello']);
